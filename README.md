## 🎯 프로젝트 개요

### 배경

- **현재 상황**: 사내 시스템 + 이메일 + MS Teams 혼용으로 정보 파편화
- **동시 진행**: 신규 개발 5개 + 설계 변경 주 5건
- **제품군**: 체성분 분석기, 혈압계, 가정용, FRA, 신장계 등
- **규모**: 50-100명 실무자, 9개 부서, 부서당 3-5명 (대부분 겸직)

### 적용 범위

- ✅ 11단계 개발 프로세스 (발의 검토 → 영업이관)
- ✅ 설계 변경 프로세스 (경미/중간/대규모 구분)
- ✅ 9개 부서: 개발, 품질, 영업, 제조, 구매, CS, 경영관리, 디자인연구소, 인증
- ✅ 외부 협력업체 (제한적 접근)

## 📊 현황 분석

### 주요 문제점

#### 🔴 1순위: 진행 상황 파악의 어려움
- 여러 도구에 정보 분산
- 현재 상태 파악에 **평균 30분 소요**
- 프로젝트별, 부서별 진행률 불명확

#### 🔴 2순위: 역할과 책임의 불명확함
- 누가 무엇을 언제까지 해야 하는지 불명확
- 겸직자 많아 역할 혼란

#### 🔴 3순위: 정보 공유 누락 (가장 심각)
- **설계 변경 시 관련 부서에 공유 안 됨**
- 영업/품질팀 요청 → 제조/구매/인증팀 미전달
- ERP 업데이트, 도면 버전 관리, 인증 재검토 누락

#### 기타 문제
- ⚠️ 일정 지연 조기 감지 불가
- ⚠️ 불필요한 미팅 과다
- ⚠️ 문서/파일 관리 체계 부족
- ⚠️ 과거 데이터 분석 어려움

## 🎯 목표 및 성공 기준

### 프로젝트 목표

> **"진행 상황 파악 시간 90% 단축 및 부서 간 소통 오류 제로화"**

### 구체적 성공 기준

#### ✅ 진행 상황 파악 시간 90% 단축
- **현재**: 30분 (여러 시스템 확인 + 이메일 검색 + 담당자 문의)
- **목표**: 3분 (대시보드 접속 즉시 확인)

#### ✅ 부서 간 소통 오류 감소
- 설계 변경 시 관련 부서 자동 알림 → **누락 제로**
- 모든 의사소통 시스템 기록 → 증거 보존

#### ✅ 불필요한 미팅 최소화
- 진행 상황 확인 미팅 → 시스템으로 대체
- 필수 미팅만 대면 (이슈 해결, 게이트 승인)

#### ✅ 일정 지연 조기 감지
- Red/Yellow/Green 상태로 위험 시각화
- 마감 임박 시 자동 알림

### 정량적 목표 (6개월 후)

| 항목 | 현재 | 목표 |
|------|------|------|
| 프로젝트 지연율 | - | 30% 감소 |
| 미팅 시간 | - | 40% 단축 |
| 설계 변경 누락 | 주 1건 | 제로 |
| 사용자 만족도 | - | 80% 이상 |

## 👥 사용자 정의

### 역할 구분

시스템 사용자는 **4가지 주요 역할**로 구분 (한 사람이 여러 역할 겸직 가능)

#### 1. 실무자 (Worker)
- **인원**: 50-100명
- **소속**: 9개 부서 각 3-5명
- **주요 업무**:
  - 체크리스트 작업 수행 및 완료 체크
  - 작업 진행률 업데이트
  - 파일/문서 업로드
  - 이슈 등록 및 코멘트 작성
  - 타 부서 협조 요청
- **디바이스**: PC + 모바일

#### 2. 부서 관리자 (Department Manager)
- **인원**: 약 18명 (부서당 2명)
- **주요 업무**:
  - 실무자 작업 검토 및 승인/반려
  - 부서 내 작업 진행 상황 모니터링
  - 지연 작업 파악 및 리소스 재배치
  - 이슈 검토 및 에스컬레이션
  - 게이트 미팅 참석
- **디바이스**: PC + 모바일 (승인용)

#### 3. 프로세스 관리자 / PM (Process Manager)
- **인원**: 프로젝트당 1명 (약 5-10명)
- **주요 업무**:
  - 전체 프로세스 진행 상황 모니터링
  - 부서 간 조율 및 충돌 해결
  - 게이트 미팅 주관 및 통과 여부 결정
  - 크리티컬 패스 관리
  - 경영진 보고 및 의사결정 지원
- **디바이스**: 주로 PC

#### 4. 일정 관리자 / Scheduler
- **인원**: 1-2명 (PM 겸직 가능)
- **주요 업무**:
  - 마스터 일정 수립 및 관리
  - 지연 위험 조기 감지 및 대응
  - 리마인더 관리
  - 일정 조정 및 시뮬레이션
  - 일정 준수율 리포팅
- **디바이스**: 주로 PC

#### 5. 외부 협력업체
- **접근 범위**: 자기가 참여하는 프로젝트만 (보안 매우 높음)
- **주요 업무**:
  - 프로젝트 진행 상황 확인
  - 설계 변경 사항 확인
  - 파일/도면 다운로드
  - 코멘트/질문 등록

### 부서별 사용자 (9개 부서)

| 부서 | 역할 | 주요 산출물 |
|------|------|------------|
| **개발팀** | 제품 설계 및 구현 | NABC, 스펙, eBOM, 도면, 기술문서 |
| **품질팀** | 신뢰성 검증 및 품질 보증 | 신뢰성 테스트 결과, 김구현 디자인리뷰 |
| **영업팀** | 시장 분석 및 판매 전략 | 시장 조사, 가격 전략, 판매 계획 |
| **제조팀** | 생산 공정 및 양산 준비 | 공정 계획, 시생산 결과 |
| **구매팀** | 부품 조달 및 공급망 관리 | 부품 계약, mBOM |
| **CS팀** | 고객 서비스 준비 | 서비스 플랜, A/S 매뉴얼 |
| **경영관리팀** | 원가 및 수익성 관리 | 원가 분석, ROI 보고서 |
| **디자인연구소** | UX/UI 및 외관 디자인 | 디자인 컨셉, UI/UX 설계 |
| **인증팀** | 각국 규제 및 인증 대응 | 인증 전략, 인증 완료 확인서 |

## 🚀 핵심 기능

### Phase 1: MVP (최우선 기능) - 2개월

#### 1. 역할별 대시보드 ⭐⭐⭐
**실무자 대시보드**
- 📋 오늘 할 일 (Today's Tasks) - 우선순위 ①
- ⏳ 승인 대기 중 (Pending Approval) - 우선순위 ②
- 📊 내 프로젝트 진행률 - 우선순위 ③
- 📢 최근 알림

**부서 관리자 대시보드**
- 👥 팀원 작업 현황
- ✅ 승인 대기 목록
- 🚨 지연 위험 (Red/Yellow/Green)
- 📊 부서 KPI

**PM 대시보드**
- 🎯 전체 프로젝트 현황
- 📈 설계 변경 현황
- ⚠️ 크리티컬 이슈
- 📅 다가오는 게이트 미팅

**일정 관리자 대시보드**
- 🚨 지연 위험
- ⏰ 마감 임박 작업
- 📊 일정 준수율
- 📅 간트 차트 요약

#### 2. 체크리스트 관리 시스템 ⭐⭐⭐

**워크플로우**
```
프로젝트 생성 
  → 11단계 × 9부서 체크리스트 자동 생성
  → 실무자: 작업 완료 체크 + 파일 첨부
  → 부서 관리자: 검토 후 승인/반려
  → 반려 시: 사유 코멘트 + 재작업 → 재승인 요청
  → 승인 시: 완료 처리 + 다음 담당자 자동 알림
```

**특징**
- 단계별, 부서별 자동 배정
- 진행률 실시간 업데이트
- 파일 첨부 및 버전 관리
- 코멘트 및 협업 기능

#### 3. 자동 알림 시스템 (Teams 연동) ⭐⭐⭐

**알림 시나리오**

| 시나리오 | 트리거 | 대상 | 타이밍 |
|---------|--------|------|--------|
| 작업 배정 | 프로젝트 시작/단계 진행 | 실무자 | 즉시 |
| 마감 임박 | 마감일 기준 | 실무자 | 3일 전, 1일 전, 당일 |
| 승인 요청 | 작업 완료 체크 | 부서 관리자 | 즉시 |
| 지연 발생 | 마감일 경과 | 담당자 + 관리자 + PM | 매일 |
| 의존 작업 완료 | 선행 작업 승인 | 후행 작업 담당자 | 즉시 |
| 설계 변경 | 변경 요청 등록 | 관련 모든 부서 | 즉시 (읽음 확인 필수) |

**Teams 알림 예시**
```
🔔 새 작업이 배정되었습니다

프로젝트: 신제품 혈압계 개발
단계: 2단계 - 기획 검토
작업: 스펙 정리 및 분석 완료
마감일: 2026-02-15 (7일 남음)

[작업 확인하기] [일정 추가]
```

#### 4. 부서 간 자동 공유 시스템 ⭐⭐⭐

**설계 변경 공유 프로세스**
```
1. 요청자(영업/품질) 변경 요청 등록
2. 시스템이 영향 부서 자동 추천 (AI + 수동 조정)
3. 규모별 프로세스 자동 생성 (경미/중간/대규모)
4. 모든 관련 부서에 Teams 알림 (읽음 확인 필수)
5. 각 부서 확인 여부 실시간 추적 (✅/⏳)
6. 미확인 부서에 1일 후 리마인더 자동 발송
7. 전체 확인 완료 시 PM에게 알림
```

**특징**
- ✅ 누락 제로 보장
- ✅ 확인 추적 시각화
- ✅ 자동 리마인더
- ✅ 변경 이력 추적

#### 5. 파일/문서 관리 ⭐⭐

- 프로젝트별, 단계별 폴더 자동 생성
- 파일 업로드/다운로드
- 버전 관리
- 검색 기능
- Firebase Storage 활용

### Phase 2: 고도화 기능 - 3개월

#### 6. 설계 변경 관리 상세

**미팅 전 준비도 자동 체크**

- 미완료 항목 자동 감지

- 미완료 부서에 독촉 알림

**미팅 진행**
- 참석자 자동 기록

- 결정 사항 입력 (승인/조건부 승인/보류/반려)

#### 8. 일정 관리 (간트 차트)

**기능**
- 프로젝트별 타임라인 시각화

- 크리티컬 패스 자동 표시
- 드래그 앤 드롭으로 일정 조정
- 지연 시 전체 일정 자동 재계산

**지연 감지 시스템**
```
매일 자동 체크
  → 마감 3일 전: 1차 알림
  → 마감 1일 전: 2차 알림 (강조)
  → 마감 당일: 3차 알림 (부서장 포함)
  → 마감일 경과: Red 상태 전환 + 매일 알림 + PM 에스컬레이션
  → 일정 재계산 및 영향도 분석
```

#### 10. 협업 기능

## 🛠 기술 스택

### Firebase 서비스 활용

```
사용자 (웹/모바일)
    ↓
Firebase App Hosting (웹 애플리케이션 배포)
    ↓
Firebase Authentication (로그인/역할별 권한)
    ↓
Firebase Firestore (실시간 데이터베이스)
    ↓
Firebase Storage (파일 저장소)
    ↓
Firebase Cloud Functions (서버리스 백엔드)
```

#### Firebase App Hosting
- **역할**: 웹 애플리케이션 배포 및 호스팅
- **활용**: 대시보드 시스템을 인터넷에 배포하여 팀원 접속 가능

#### Firebase Authentication
- **역할**: 사용자 인증 및 권한 관리
- **활용**: 
  - 이메일/비밀번호 로그인
  - 역할별 접근 제어 (실무자/관리자/PM/외부)
  - Google SSO (선택)

#### Firebase Firestore (Data Connect)
- **역할**: 실시간 NoSQL 데이터베이스
- **활용**:
  - 프로젝트 정보, 체크리스트, 설계 변경, 이슈 등 저장
  - 실시간 동기화 (새로고침 불필요)
  - 복잡한 쿼리 및 필터링

**컬렉션 구조 예시**
```
/projects/{projectId}
  - 프로젝트 기본 정보
  /checklists/{checklistId}
    - 체크리스트 항목
  /changeRequests/{changeId}
    - 설계 변경 요청
  /gates/{gateId}
    - 게이트 미팅 정보
  /issues/{issueId}
    - 이슈 관리

/users/{userId}
  - 사용자 정보, 역할, 부서

/notifications/{notificationId}
  - 알림 기록
```

#### Firebase Storage
- **역할**: 파일 저장소
- **활용**:
  - 설계 문서, 도면, 보고서, 이미지 등 저장
  - 프로젝트별, 단계별 폴더 구조
  - 버전 관리

#### Firebase Cloud Functions
- **역할**: 서버리스 백엔드 로직
- **활용**:
  - 자동 알림 발송 (Teams API 연동)
  - 마감일 체크 및 지연 감지 (매일 자동 실행)
  - 의존성 작업 완료 시 다음 담당자 알림
  - AI 기반 영향 부서 자동 추천
  - 리포트 자동 생성

**Cloud Function 예시**
```javascript
// 작업 완료 시 승인 요청 알림
exports.onTaskCompleted = functions.firestore
  .document('projects/{projectId}/checklists/{checklistId}')
  .onUpdate((change, context) => {
    const newData = change.after.data();
    if (newData.status === 'completed') {
      sendTeamsNotification(newData.managerId, {
        title: '승인 요청',
        message: `${newData.worker}님이 작업을 완료했습니다.`
      });
    }
  });

// 매일 자동 실행: 마감일 체크
exports.scheduledDeadlineCheck = functions.pubsub
  .schedule('every day 09:00')
  .timeZone('Asia/Seoul')
  .onRun(async (context) => {
    // 마감일 체크 로직
  });
```

### Firebase를 선택한 이유

#### 1. 빠른 개발
- 서버 구축 불필요
- 백엔드 코드 작성 최소화
- 인프라 관리 불필요

#### 2. 실시간 동기화
```
실무자가 체크 완료 → Firestore 즉시 업데이트 
→ 관리자 화면 실시간 반영 (새로고침 불필요)
```

#### 3. 비용 효율적
- 무료 플랜으로 시작 가능
- 사용량만큼만 과금
- 별도 서버 운영 비용 없음

#### 4. 확장성
- 사용자 증가 시 자동 확장
- 여러 프로젝트 동시 관리 가능

#### 5. 보안
- Google 수준의 보안
- 역할별 접근 권한 설정 (Security Rules)
- 데이터 암호화
