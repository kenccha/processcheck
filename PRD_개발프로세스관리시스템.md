# PRD: 개발 프로세스 관리 시스템
**버전**: 1.0
**작성일**: 2026-02-22
**상태**: 검토 중

---

## 문제 정의 (Problem Statement)

전자제품 개발 조직(체성분 분석기, 혈압계, 가정용 기기, FRA, 신장계 등)에서 9개 부서 50~100명의 실무자가 **사내 ERP, 이메일, MS Teams를 혼용**하면서 정보가 파편화되고 있다. 현재 프로젝트 상태를 파악하는 데 평균 30분이 소요되고, 설계 변경(ECR) 발생 시 관련 부서에 주 평균 1건씩 공유가 누락되어 재작업·일정 지연·비용 증가로 이어지고 있다. 이 문제를 해결하지 않으면 동시 진행 중인 신규 개발 5개 + 주간 설계 변경 5건 규모가 더 늘어날수록 운영 비용과 오류 빈도가 비례해서 증가한다.

---

## 목표 (Goals)

1. **상태 파악 시간 90% 단축**: 프로젝트 현황 확인 시간을 평균 30분 → 3분 이내로 단축 (Phase 1 출시 후 1개월 내)
2. **설계 변경 공유 누락 제로화**: ECR 발생 시 관련 부서 자동 알림으로 누락 0건 달성 (Phase 1 출시 직후)
3. **불필요한 진행 상황 확인 미팅 40% 감소**: 대시보드 도입으로 상태 확인 목적 미팅 주 3회 → 1.5회로 축소 (Phase 2 출시 후 분기 내)
4. **프로젝트 일정 지연율 30% 감소**: Red/Yellow 상태 자동 감지 및 조기 대응 체계로 달성 (Phase 2 출시 후 6개월 내)
5. **사용자 만족도 80% 이상**: 시스템 출시 후 월간 설문 기준 (Phase 2 정식 출시 후 3개월 내)

---

## 범위 외 항목 (Non-Goals)

- **ERP 데이터 마이그레이션 자동화**: 기존 시스템 데이터 이전은 이번 범위 포함하지 않음. 새 프로젝트부터 신규 시스템에 입력하는 방식으로 전환. 마이그레이션 자동화는 운영 안정화 이후 별도 검토.
- **ERP 시스템과의 실시간 양방향 연동**: ERP 연동은 Phase 3 이후 검토 사항. 현재는 수동 입력 또는 단방향 참조로 운영.
- **AI 기반 일정 자동 최적화**: 영향 부서 자동 추천(Phase 3)은 포함되나, AI가 일정을 자동으로 재조정하는 기능은 이번 로드맵 외.
- **외부 협력업체 프로젝트 생성 권한**: 외부 협력업체는 읽기·코멘트 권한만 부여. 프로젝트 생성·수정·승인은 내부 임직원 전용.
- **타사 프로젝트 관리 도구(Jira, Asana 등) 대체**: 이 시스템은 사내 11단계 개발 프로세스에 특화된 맞춤형 도구로, 범용 PM 툴 전환을 목표로 하지 않음.

## 사용자 스토리 (User Stories)

### 실무자 (Worker) — 50~100명

- 실무자로서, 오늘 내가 해야 할 작업을 시스템에 접속하는 즉시 확인하고 싶다. 그래야 여러 채널을 뒤지지 않고 집중할 수 있다.
- 실무자로서, 작업을 완료하면 버튼 한 번으로 부서 관리자에게 승인 요청을 보내고 싶다. 그래야 이메일로 별도 보고하는 수고를 덜 수 있다.
- 실무자로서, 마감이 임박했을 때 Teams로 자동 알림을 받고 싶다. 그래야 마감을 놓치는 실수를 방지할 수 있다.
- 실무자로서, 관련 파일(도면, 보고서)을 작업 항목에 직접 첨부하고 싶다. 그래야 파일을 찾기 위해 공유 드라이브를 별도로 뒤질 필요가 없다.

### 부서 관리자 (Department Manager) — 약 18명

- 부서 관리자로서, 팀원별 작업 완료 현황과 지연 위험을 한 화면에서 보고 싶다. 그래야 개별적으로 물어보는 시간을 줄일 수 있다.
- 부서 관리자로서, 실무자의 작업 완료 요청을 모바일에서도 승인하거나 반려할 수 있고 싶다. 그래야 외근 중에도 워크플로우를 막지 않을 수 있다.
- 부서 관리자로서, 반려 시 사유를 코멘트로 남기고 싶다. 그래야 실무자가 왜 재작업이 필요한지 명확히 이해할 수 있다.

### 프로세스 관리자 / PM — 5~10명

- PM으로서, 담당 프로젝트 전체의 11단계 진행률을 부서별로 한 눈에 보고 싶다. 그래야 지연 위험이 있는 부서를 선제적으로 조율할 수 있다.
- PM으로서, 게이트 미팅 전 각 부서의 체크리스트 완료율을 자동 집계한 준비도 리포트를 받고 싶다. 그래야 미팅 준비 시간을 줄이고 미완료 부서에 사전 요청할 수 있다.
- PM으로서, 크리티컬 이슈 발생 시 즉시 알림을 받고 싶다. 그래야 이슈가 확대되기 전에 개입할 수 있다.

### 일정 관리자 (Scheduler) — 1~2명

- 일정 관리자로서, 마감이 임박하거나 지연된 작업 목록을 Red/Yellow로 시각화해 보고 싶다. 그래야 우선순위에 따라 리소스를 빠르게 재배치할 수 있다.
- 일정 관리자로서, 특정 작업 지연 시 이후 연결된 작업 일정이 어떻게 영향받는지 자동 재계산 결과를 보고 싶다. 그래야 전체 일정 영향을 수작업 없이 파악할 수 있다.

### 외부 협력업체 — 제한 접근

- 외부 협력업체로서, 내가 참여하는 프로젝트의 최신 설계 변경 사항과 도면을 시스템에서 직접 확인하고 싶다. 그래야 이메일로 별도 요청하지 않고 최신 정보를 얻을 수 있다.

## 요구사항 (Requirements)

### P0 — 필수 (Must-Have, Phase 1 MVP)

**역할별 대시보드**
- 실무자, 부서 관리자, PM, 일정 관리자별 맞춤형 대시보드 화면 제공
- 접속 즉시 3분 이내 핵심 정보 파악 가능
- 수락 기준: 각 역할 사용자가 대시보드 진입 후 추가 클릭 없이 현재 프로젝트 상태와 당일 할 일 확인 가능

**체크리스트 자동 생성 및 관리**
- 프로젝트 생성 시 11단계 × 9부서 체크리스트 자동 생성
- 실무자의 완료 체크 → 부서 관리자 승인/반려 워크플로우
- 반려 시 사유 코멘트 필수 입력, 재작업 후 재승인 요청
- 수락 기준: 프로젝트 생성 후 5분 이내 전체 체크리스트 자동 배정 완료

**MS Teams 자동 알림**
- 작업 배정, 마감 3일/1일/당일 전, 승인 요청, 지연 발생, 설계 변경 시 자동 알림
- 설계 변경 알림은 수신 확인(읽음 확인) 필수
- 수락 기준: 트리거 이벤트 발생 후 5분 이내 Teams 알림 도착

**설계 변경(ECR) 등록 및 부서 공유**
- 변경 요청 등록 → 영향 부서 수동 선정(AI 추천은 Phase 3) → 자동 알림 발송
- 각 부서 확인 여부 실시간 추적 (✅/⏳ 시각화)
- 미확인 부서 1일 후 자동 리마인더
- ECR 번호 자동 생성 (ECR-YYYY-NNN)
- 수락 기준: ECR 등록 후 모든 지정 부서가 알림 수신, 누락 0건

**Firebase 기반 인증 및 역할 관리**
- 이메일/비밀번호 로그인, 역할별 접근 제어 (RBAC)
- 외부 협력업체: 지정 프로젝트 읽기·코멘트만 허용
- 수락 기준: 역할 외 메뉴·데이터 접근 시 권한 오류 반환

**파일 업로드 및 관리**
- 작업 항목에 파일 직접 첨부 (도면, 보고서 등)
- 프로젝트별, 단계별 자동 폴더 구조
- 수락 기준: 파일 업로드 후 동일 작업 페이지에서 다운로드 가능

**모바일 반응형 UI**
- 승인/반려, 알림 확인 기능 모바일에서 사용 가능
- 수락 기준: iOS/Android 브라우저에서 핵심 기능 오류 없이 동작

---

### P1 — 권장 (Nice-to-Have, Phase 2)

**게이트 미팅 관리**
- 미팅 전 부서별 체크리스트 완료율 자동 집계 및 준비도 리포트 생성
- 미팅 결과(승인/조건부/보류/반려) 기록 및 액션 아이템 자동 배정
- 수락 기준: 게이트 미팅 24시간 전 자동으로 준비도 리포트 PM에게 발송

**간트 차트 (일정 시각화)**
- 프로젝트별 타임라인 및 의존성 관계 시각화
- 크리티컬 패스 자동 표시
- 드래그 앤 드롭으로 일정 조정 시 후행 작업 자동 재계산
- 수락 기준: 일정 변경 후 5초 이내 전체 영향 일정 재계산 완료

**이슈 관리**
- 이슈 등록, 심각도 설정(High/Medium/Low), 담당자 배정
- 상태 관리 (Open → In Progress → Resolved) 및 히스토리 기록

**협업 기능**
- 작업 항목별 코멘트/토론, @멘션으로 특정인 호출
- 타 부서 검토/협조 요청 워크플로우

**고급 지연 감지**
- Red/Yellow/Green 상태 시각화, 지연 시 매일 PM 에스컬레이션
- 수락 기준: 마감일 경과 후 24시간 이내 PM에게 자동 에스컬레이션

### P2 — 미래 검토 (Future Considerations, Phase 3)

**리포팅 및 분석 대시보드**: 프로젝트별 진행 보고서 자동 생성, 지연 원인 통계, KPI 대시보드

**AI 기반 영향 부서 추천**: ECR 등록 시 과거 유사 변경 이력 분석으로 관련 부서 자동 추천

**일정 시뮬레이션 (What-if 분석)**: 가상 시나리오로 일정 영향도 사전 분석

**외부 시스템 연동**: ERP 단방향 연동, 다국어 지원 (한국어/영어)

**모바일 앱화**: PWA 또는 네이티브 앱으로 고도화
