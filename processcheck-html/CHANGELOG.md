## 2026-03-12 (세션 2) — UX 개선 및 편집 기능

### 빠른 작업 추가 입력 UX 변경
- `@담당자 #부서 !긴급 작업내용 ~마감일` 구문 방식 → 드롭다운 기반 폼으로 변경
- 작업 제목 입력 + 단계/부서/중요도 셀렉트 + 마감일 date picker

### 스케줄 탭 타임라인 개선
- 월별 라벨 + 세로 그리드 추가
- "오늘" 빨간 마커 + 날짜 뱃지
- 경과/잔여일 요약
- Phase별 진행률 바 + 완료/전체 표시
- 게이트 승인 상태 아이콘

### Phase 체크리스트 분리
- 단일 목록 → "작업 체크리스트" (📝 작업 단계) + "승인 체크리스트" (🔒 게이트 단계) 분리
- 부서별 그룹핑

### Phase 기간 편집 기능
- Phase 카드 헤더에 ✏️ 편집 버튼 추가
- 인라인 시작일/종료일 date picker
- "적용" 시 해당 Phase 전체 작업 마감일 균등 분배

### 날짜 형식 통일
- 모든 페이지의 날짜 형식을 `YY/MM/DD`로 통일
- `formatDate()`, `formatDateTime()` 중앙 집중화 (`utils.js`)
- 리뷰 패널 설정(⚙️)에서 사용자 날짜 형식 선택 가능
- 지원 형식: YY/MM/DD, YYYY/MM/DD, YYYY-MM-DD, MM/DD/YYYY, YYYY.MM.DD

### Phase 카드 내 회의록 인라인 표시
- 각 Phase 카드 본문에 최근 3건 회의록 표시

### CSS 둥근 디자인 통일
- `--radius-sm` ~ `--radius-2xl`, `--radius-full` CSS 변수 추가/증가
- 승인 버튼 (승인/반려/초기화) 둥글게 + hover 효과
- `.gate-section`, `.gate-note`, `.phase-gate-btn` 등 둥글게 통일

### 버튼 디자인 개선
- 승인/반려/초기화 버튼: `border-radius: var(--radius-xl)`, padding 증가
- hover 시 `translateY(-1px)` + transition 효과

### 리스트 뷰 제거

- fallback을 Phase 뷰로 변경

### 매트릭스 뷰 승인 상태 표시
