and register both in firebase.json so they deploy with the hosting.

require Firebase Auth. This closes the critical gap where any HTTP client with the Firebase config
(public in client code) could write arbitrary data to any collection.

storage.rules, updated firebase.json with storage section.

<task type="auto">
  <name>Task 1: Harden firestore.rules — require isAuthenticated() for all collection writes</name>
  <files>firestore.rules</files>
  <action>
    Rewrite firestore.rules completely. The new rules must:

    1. Keep `isAuthenticated()` helper function as-is.
    2. Remove `isStoredUser()` helper — it returns `true` unconditionally and was a placeholder.
    3. For EVERY collection: `allow read: if true;` (keep reads open — customer portal and
       demo users need reads, and role data lives in Firestore not Auth tokens).
    4. For EVERY collection: `allow write: if isAuthenticated();` (no more `|| true` bypasses).
    5. Add explicit rules for activityLogs (currently missing — would silently deny writes in prod).
    6. Check if feedbacks collection has a rule; if not, add one.

    The complete replacement content for firestore.rules:

    ```
    rules_version = '2';

    service cloud.firestore {
      match /databases/{database}/documents {

        // ─── Helper Functions ─────────────────────────────────────────────────
        function isAuthenticated() {
          return request.auth != null;
        }

        // ─── Users ────────────────────────────────────────────────────────────
        // Read open: user list needed for task assignment dropdowns
        match /users/{userId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Projects ─────────────────────────────────────────────────────────
        match /projects/{projectId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Checklist Items ──────────────────────────────────────────────────
        match /checklistItems/{itemId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Template Stages ──────────────────────────────────────────────────
        match /templateStages/{stageId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Template Departments ─────────────────────────────────────────────
        match /templateDepartments/{deptId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Template Items ───────────────────────────────────────────────────
        match /templateItems/{itemId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Notifications ────────────────────────────────────────────────────
        match /notifications/{notifId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Change Requests ──────────────────────────────────────────────────
        match /changeRequests/{requestId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Customers ────────────────────────────────────────────────────────
        match /customers/{customerId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Launch Checklists ────────────────────────────────────────────────
        match /launchChecklists/{checklistId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Portal Notifications ─────────────────────────────────────────────
        match /portalNotifications/{notifId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Reviews ──────────────────────────────────────────────────────────
        match /reviews/{reviewId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

        // ─── Activity Logs ────────────────────────────────────────────────────
        // Written by firestore-service.js; was missing from rules (implicit deny)
        match /activityLogs/{logId} {
          allow read: if isAuthenticated();
          allow write: if isAuthenticated();
        }

        // ─── Feedbacks ────────────────────────────────────────────────────────
        // Written by docs/deliverables/js/feedback-system.js
        match /feedbacks/{feedbackId} {
          allow read: if true;
          allow write: if isAuthenticated();
        }

      }
    }
    ```

    IMPORTANT: The existing `reviews` rule was already correct — preserve it.
    Do NOT add role-based checks (observer/manager/worker) — role data is in Firestore and
    `request.auth.uid` does not match any Firestore user document ID (auto-generated IDs).
    Role enforcement remains client-side only.

<task type="auto">
  <name>Task 2: Create storage.rules and register it in firebase.json (SEC-02)</name>
  <files>storage.rules, firebase.json</files>
  <action>
    **Create storage.rules (new file at processcheck-html/storage.rules):**

    ```
    rules_version = '2';
    service firebase.storage {
      match /b/{bucket}/o {
        match /{allPaths=**} {
          allow read, write: if request.auth != null;
        }
      }
    }
    ```

    This requires Firebase Auth for all Storage operations. Unauthenticated uploads and downloads
    are rejected. MS OAuth users (who have a real Firebase Auth token) can upload/download.
    Demo card users on localhost will not be able to upload — this is acceptable since file upload
    is Phase 4 work and demo usage is localhost-only by Phase 1 design.

    **Update firebase.json — add storage section:**

    Read current firebase.json first. The current content is hosting-only. Add a "storage" key
    at the top level alongside "hosting":

    ```json
    {
      "hosting": {
        ...existing hosting config unchanged...
      },
      "storage": {
        "rules": "storage.rules"
      }
    }
    ```

    Keep all existing hosting config (public, ignore, headers) exactly as-is. Only add the
    "storage" key.

    **Deploy both rule sets:**

    After writing the files, run from /Users/injooncha/processcheck/processcheck-html/:

    ```bash
    firebase deploy --only firestore:rules,storage
    ```

    This deploys both Firestore rules (from Task 1) and Storage rules together.
    If the deploy fails due to missing Firebase CLI auth, note the error in the summary —
    the file content is correct and the deploy can be retried by the user.
  </action>
