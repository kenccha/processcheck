  <action>
    Add session TTL enforcement to js/auth.js. Three changes:

    1. Add constants after STORAGE_KEY:
       ```javascript
       const SESSION_TTL = 24 * 60 * 60 * 1000; // 24 hours in ms
       ```

    2. In login() — add loginAt to the session object before localStorage.setItem:
       Change: `const sessionUser = { ...user, role };`
       To:     `const sessionUser = { ...user, role, loginAt: Date.now() };`

    3. In loginWithMicrosoft() — add loginAt in BOTH places where msUser is built and saved:
       - Existing user branch: `const msUser = { ...existingUser, authProvider: "microsoft", loginAt: Date.now() };`
       - New user branch:      `const msUser = { ...newUser, authProvider: "microsoft", loginAt: Date.now() };`

    4. In completeRegistration() — add loginAt:
       Change: `const msUser = { ...newUser, authProvider: "microsoft" };`
       To:     `const msUser = { ...newUser, authProvider: "microsoft", loginAt: Date.now() };`

    5. Modify guardPage() to check expiry:
       ```javascript
       export function guardPage() {
         const user = getUser();
         if (!user) {
           window.location.href = "index.html";
           return null;
         }
         if (user.loginAt && Date.now() - user.loginAt > SESSION_TTL) {
           logout();
           return null;
         }
         return user;
       }
       ```
       Note: logout() is async but we do not await it here — it calls window.location.href which
       triggers immediately. This matches existing patterns in the codebase.

    6. Add and export startSessionWatcher() after guardPage():
       ```javascript
       export function startSessionWatcher() {
         const INTERVAL = 5 * 60 * 1000; // check every 5 minutes
         setInterval(() => {
           const user = getUser();
           if (!user || (user.loginAt && Date.now() - user.loginAt > SESSION_TTL)) {
             logout();
           }
         }, INTERVAL);
       }
       ```

    Do NOT add startSessionWatcher() calls to individual page files — that is handled in Task 2
    via components.js initNav() which every protected page already calls.
