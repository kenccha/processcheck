GTD — Getting Things Done

David Allen의 스트레스 없는 생산성 시스템

"머리는 아이디어를 떠올리는 곳이지, 저장하는 곳이 아니다." — David Allen
💡
GTD 핵심 개념

GTD의 핵심은 단순하다: 머릿속에 있는 모든 것을 꺼내서 신뢰할 수 있는 시스템에 넣고, 그 시스템을 믿고 행동하는 것.


왜 필요한가?

  • 열린 루프(Open Loop) 제거 — "아, 그거 해야 하는데..." 하는 것들이 머릿속에 있으면 인지 부하가 쌓인다
  • 다음 행동(Next Action) 명확화 — "프로젝트 진행" 같은 모호한 할 일이 아니라 "김 부장에게 견적서 이메일 보내기" 같은 구체적 행동
  • 적절한 순간에 적절한 일 — 지금 이 상황(장소, 에너지, 시간)에 맞는 일을 선택
⚙️
5단계 워크플로우
1

Capture

수집

머릿속의 모든 것을 Inbox로 꺼낸다. 이메일, 메모, 슬랙, 구두 요청 전부.

2

Clarify

명확화

하나씩 판단한다. 행동이 필요한가? 2분 안에 되나? 위임? 연기?

3

Organize

정리

판단 결과를 적절한 리스트에 배치. Next Actions, Projects, Waiting For 등.

4

Reflect

검토

주간 리뷰로 전체 시스템을 점검. 빠진 것, 업데이트할 것 확인.

5

Engage

실행

시스템을 믿고 지금 할 수 있는 일을 한다. 상황, 시간, 에너지 기준.

🔀
Clarify 판단 흐름도 (가장 중요한 단계)
핵심 규칙: 2분 안에 할 수 있는 일은 지금 바로 해라. 리스트에 적는 시간이 더 오래 걸린다.
📋
GTD가 관리하는 리스트들

✅ Next Actions

지금 당장 할 수 있는 구체적 행동들. 맥락(@컴퓨터, @전화, @외출)으로 분류하면 더 좋다.

📋 Projects

2개 이상의 행동이 필요한 결과물. "보고서 작성", "신제품 출시" 등. 각 프로젝트마다 최소 1개의 Next Action이 있어야 한다.

⏳ Waiting For

다른 사람에게 위임했거나 기다리는 것들. 누구에게, 언제 요청했는지 기록.

💭 Someday / Maybe

지금은 아니지만 언젠가 하고 싶은 것들. 주간 리뷰에서 정기적으로 재검토.

📅 Calendar

특정 날짜/시간에 해야 하는 것만. "하면 좋겠다" 수준은 넣지 않는다. 캘린더는 신성하게.

📂 Reference

행동은 필요 없지만 나중에 참고할 정보. 회의록, 매뉴얼, 계약서 등.

📧
실전 예시: 이메일 10통 처리하기

월요일 아침, 받은편지함에 쌓인 10통의 이메일을 GTD로 처리하는 과정:

마케팅팀 박과장
"3분기 브로슈어 시안 확인 부탁드립니다"
⚡ 바로 한다 (2분)
→ 시안 훑어보고 "확인했습니다, 진행해주세요" 답장 (1분 소요)
대리점 A (고객)
"제품 X 납기일 변경 가능한지 문의"
👤 위임
→ 영업팀 김대리에게 전달, Waiting For에 "김대리 — 대리점A 납기 회신 (3/11)" 기록
인사팀
"연차 사용 현황 안내"
📂 참고자료
→ "인사/연차" 폴더로 이동. 행동 불필요.
팀장
"신규 프로젝트 킥오프 미팅 3/15(금) 14시"
📅 캘린더
→ 3/15 14시 캘린더에 등록. Next Action: "킥오프 미팅 전 현 프로젝트 현황 정리"
외부 세미나 안내
"AI 트렌드 세미나 4월 개최 안내"
💭 언젠가/아마도
→ 관심은 있지만 지금은 바쁘다. Someday/Maybe 리스트에 추가.
쇼핑몰 뉴스레터
"봄맞이 특가 세일 안내"
🗑️ 삭제
→ 행동 불필요, 참고가치 없음. 즉시 삭제 또는 구독 해지.
품질팀 이과장
"테스트 결과 보고서 리뷰 요청"
📝 Next Action
→ 2분 안에 안 됨. Next Actions에 "@컴퓨터 — 품질 테스트 보고서 리뷰 후 피드백" 추가
경영기획실
"2026 하반기 사업계획 수립 협조 요청"
📋 프로젝트
→ 여러 단계 필요. Projects에 "하반기 사업계획 수립" 등록. 첫 Next Action: "지난 사업계획 파일 찾기"
IT팀
"비밀번호 변경 안내 (90일 주기)"
⚡ 바로 한다 (2분)
→ 지금 바로 비밀번호 변경. 완료 후 이메일 삭제.
디자인팀 최주임
"UI 와이어프레임 v2 공유합니다"
📝 Next Action
→ Next Actions: "@컴퓨터 — 와이어프레임 v2 검토, 코멘트 달기" + Waiting For: "최주임 — 수정본 받기"
결과: 10통 처리 완료. 바로 한 것 2건, Next Actions 3건 추가, 위임 1건, 프로젝트 1건 등록, 캘린더 1건, 참고자료 1건, 삭제 1건. 받은편지함 = 0.
🗓️
1주일 실행 플랜 이번 주
Day 1 (월) — 셋업
  • 도구 선택: 메모앱(Notion, Obsidian), 이메일 폴더, 캘린더 중 하나. 복잡하게 시작하지 말 것
  • Inbox 만들기: 모든 생각을 던질 하나의 장소 (노트앱 페이지 하나 or 물리적 노트)
  • 리스트 4개 생성: Next Actions, Projects, Waiting For, Someday/Maybe
  • Brain Dump: 30분 동안 머릿속의 모든 미해결 건을 적는다 (최소 20개 목표)
Day 2 (화) — 이메일 정복
  • 이메일 폴더 생성: @Action, @Waiting, @Reference (3개면 충분)
  • 받은편지함 50통 처리: 위의 판단 흐름도를 보면서 하나씩 판단
  • 2분 룰 체화: 2분 안에 되는 건 바로 처리, 나머지는 리스트에 배치
  • 목표: 받은편지함 0건 (Inbox Zero) 달성
Day 3 (수) — Brain Dump 정리
  • Day 1에 적은 모든 항목을 판단 흐름도로 분류
  • 프로젝트에는 반드시 "다음 행동(Next Action)" 1개 이상 지정
  • 모호한 항목은 구체적 행동으로 변환 (예: "건강 관리" → "헬스장 등록 전화하기")
Day 4-5 (목금) — 실행 & 습관화
  • 아침 루틴: 캘린더 확인 → Next Actions 리스트 훑기 → 시작 (5분)
  • 이메일 처리: 하루 2-3회 정해진 시간에만 확인, 즉시 GTD 처리
  • 새로운 인풋 수집: 회의 중 떠오른 것, 슬랙 메시지 → 전부 Inbox로
  • 하루 끝에 Inbox 비우기 (5분)
Day 6-7 (주말) — 첫 주간 리뷰
  • 모든 Inbox 비우기: 이메일, 메모, 책상 위
  • Projects 리스트 점검: 각 프로젝트에 다음 행동이 있는지 확인
  • Waiting For 팔로업: 기다리는 건에 대해 확인 필요한 것 체크
  • Someday/Maybe 리뷰: 지금 시작할 게 있는지, 삭제할 게 있는지
  • 다음 주 캘린더 미리보기: 준비할 것 없는지 확인
  • 자기 평가: GTD가 도움이 됐나? 어디서 막혔나?

🟢 시스템 작동 여부

  • 주간 리뷰를 4회 중 몇 회 했는가?
  • Inbox가 매일 비워지고 있는가?
  • Next Actions 리스트를 매일 보고 있는가?
  • "아, 그거 깜빡했다" 순간이 줄었는가?

🔵 리스트 건강 상태

  • Projects 리스트에 Next Action 없는 건이 있는가? (있으면 문제)
  • Waiting For가 방치되고 있지 않은가?
  • Someday/Maybe에 실제로 시작한 게 있는가?
  • 참고자료를 쉽게 찾을 수 있는가?

🟡 생산성 변화

  • 이메일 처리 시간이 줄었는가?
  • 회의 후 행동이 더 명확해졌는가?
  • 마감 직전 급한 일이 줄었는가?
  • "뭘 해야 하지?" 고민 시간이 줄었는가?

🔴 개선할 점

  • 너무 복잡한 도구를 쓰고 있지 않은가?
  • 수집은 하는데 처리를 안 하고 있지 않은가?
  • 주간 리뷰를 건너뛰고 있지 않은가?
  • GTD를 "완벽하게" 하려다 오히려 부담인가?
David Allen의 조언: GTD는 "완벽하게 실행하는 것"이 아니라 "떨어졌을 때 빨리 돌아오는 것"이 핵심이다. 주간 리뷰만 꾸준히 하면 시스템은 유지된다.

📋 주간 리뷰 — 30분 투자, 1주일 통제력
GET CLEAR — 모든 Inbox 비우기 (이메일, 메모, 책상)
GET CLEAR — 새로운 항목 전부 Clarify 처리하기
GET CURRENT — Next Actions 리스트: 완료 항목 제거, 새 항목 추가
GET CURRENT — Projects 리스트: 각 프로젝트의 다음 행동 확인
GET CURRENT — Waiting For: 팔로업 필요한 건 체크, 기한 지난 건 재촉
GET CURRENT — 다음 주 캘린더 확인, 준비 사항 Next Action에 추가
GET CREATIVE — Someday/Maybe 훑기: 시작할 것? 삭제할 것?
GET CREATIVE — 새로운 아이디어나 프로젝트 떠오른 것 있으면 수집
